## 一、系统概述

### 1.1 系统定位

天命劫系统是游戏的**核心差异化系统**，融合Roguelike元素和东方"劫难"文化，为每次游戏体验带来**新鲜感**、**挑战性**和**东方宿命感**。

### 1.2 设计目标

| 目标 | 描述 | 东方元素 |
|------|------|----------|
| 增加重复可玩性 | 每次体验不同 | 天命轮回，劫难随机 |
| 制造惊喜感 | 随机事件的刺激 | 天降祥瑞/凶兆 |
| 提供策略选择 | 事件带来抉择 | 道心考验 |
| 永久进度 | 功法永久生效 | 修仙传承 |

### 1.3 系统架构

```
【天命劫系统 - MVP已实现架构】

天命劫系统
├── 天命事件系统 ✅（每3波触发，10种事件）
│   ├── 正面事件（6种）：灵气恩赐/灵石雨/阵法加持/悟道顿悟/仙符降世/妖气衰退
│   ├── 负面事件（3种）：灵气枯竭/妖气弥漫/阵法腐蚀
│   └── 中性事件（1种）：天道试炼

├── 问道探索系统 ✅（消耗灵石+冷却20秒，8种事件）
│   ├── 高收益：灵脉矿藏/悟道机缘
│   ├── 中收益：上古洞府/仙药园/困兽之斗/阵法残卷

├── 修仙功法系统 ✅（6种永久加成功法）
│   ├── 万剑诀/金刚护体/炼丹术/五行归元/驭兽心经/天劫心法

├── 渡劫系统 ✅（Lv3风劫/Lv6火劫/Lv9雷劫）
```

### 2.2 事件触发流程

```

       ↓
```

### 3.3 权重随机算法

```javascript
// 代码实现：rollDestinyEvent() 同款算法
function rollExploreEvent() {
    const totalWeight = EXPLORE_EVENTS.reduce((s, e) => s + e.weight, 0);
    let roll = Math.random() * totalWeight;
    for (const evt of EXPLORE_EVENTS) {
        roll -= evt.weight;
        if (roll <= 0) return evt;
    }
    return EXPLORE_EVENTS[0];
}
```


### 5.3 渡劫伤害计算

```
【渡劫伤害公式】

每tick伤害 = config.dmgPerTick（3/5/8）
每秒触发1次tick
灵木灵气 -= dmgPerTick

成功条件：duration秒后灵气 > 0
失败条件：灵气 ≤ 0

最低灵气需求估算：
├── 风劫(Lv3)：15秒 × 3/秒 = 45灵气（Lv3上限200 → 安全）
├── 火劫(Lv6)：20秒 × 5/秒 = 100灵气（Lv6上限500 → 安全）
└── 雷劫(Lv9)：25秒 × 8/秒 = 200灵气（Lv9上限1100 → 安全，但需注意灵气回复）
```
